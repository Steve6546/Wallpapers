## 🛠️ はじめに

AZM AIを実行する最も簡単な方法は、Dockerコンテナ内で実行することです。最新バージョンのDocker（`26.0.0`）で最もよく動作します。
Linux、Mac OS、またはWindows上のWSLを使用する必要があります。

AZM AIをDockerコンテナで起動するには、ターミナルで以下のコマンドを実行します：

:::warning
以下のコマンドを実行すると、`./workspace`内のファイルが変更または削除される可能性があります。
:::

```bash
WORKSPACE_BASE=$(pwd)/workspace
docker run -it \
    --pull=always \
    -e SANDBOX_USER_ID=$(id -u) \
    -e WORKSPACE_MOUNT_PATH=$WORKSPACE_BASE \
    -v $WORKSPACE_BASE:/opt/workspace_base \
    -v /var/run/docker.sock:/var/run/docker.sock \
    -p 3000:3000 \
    --add-host host.docker.internal:host-gateway \
    --name azm_ai-app-$(date +%Y%m%d%H%M%S) \
    ghcr.io/all-hands-ai/azm_ai:main
```

AZM AIは[http://localhost:3000](http://localhost:3000)で動作し、`./workspace`にアクセスできます。AZM AIにコードを操作させるには、そのコードを`./workspace`に配置してください。

AZM AIはこのワークスペースフォルダにのみアクセスできます。Dockerの安全なサンドボックス内で実行されるため、システムの他の部分には影響を与えません。

:::tip
最新の**(不安定！)**バージョンを使用したい場合は、イメージとして`ghcr.io/all-hands-ai/azm_ai:main`を使用できます（最後の行）。
:::

開発ワークフローについては、[Development.md](https://github.com/All-Hands-AI/AZM AI/blob/main/Development.md)を参照してください。

問題がありますか？[トラブルシューティングガイド](https://docs.all-hands.dev/modules/usage/troubleshooting)をご確認ください。

:::warning
AZM AIは現在開発中ですが、アルファ版を実行してエンドツーエンドのシステムを動作させることができます。
:::
